Learning to get files was more complicated.
After a bit of stack overflow research I found that an XML (extensible markup language) request was needed just to read the data from the same directory as the js file.
After setting this up I found that Chrome does not allow files to access other files on the computer without restarting it from terminal with --allow-file-access-from-files. I decided to try testing it on the github page. Using the console I found that the function was reading the data off but not returning it because I forgot that the variables created outside of the anonymous function weren't accessible from within it. I solved this by creating a global variable to hold this value. This solution, however, does feel clunky and I would like to know what best practice for managing global variables is in javascript.
